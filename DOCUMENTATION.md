# Sentinel-Duty AI: Technical Documentation

## 1. System Architecture
Sentinel-Duty AI is built as a real-time behavioral analytics engine designed to ingest transaction data and output vulnerability risk scores aligned with FCA 2026 Consumer Duty standards.

### Tech Stack
* **Language:** Python 3.9+
* **Frontend:** Streamlit (Real-time Dashboarding)
* **Data Processing:** Pandas / NumPy (Vectorized behavioral analysis)
* **Model Logic:** Rule-based heuristic engine with an Extensible ML Layer.

---

## 2. Core Logic & Detection Algorithms

### A. Financial Vulnerability Scoring (FVS)
The system calculates a risk score based on three primary vectors:
1. **Velocity of Spend:** Monitoring for rapid-fire transactions to gambling merchants or high-risk MCCs (Merchant Category Codes).
2. **Income-to-Expense Ratio:** Detecting "Income Shocks" where essential spending exceeds average monthly inflow.
3. **Behavioral Drift:** Identifying deviations from a user's 90-day spending baseline.

### B. The Audit Trail
To satisfy FCA requirements for **Outcome Monitoring**, every flag generated by the engine includes a JSON-serialized "Reasoning Object." This ensures that compliance officers can explain *why* a customer was flagged.

---

## 3. Data Models
The engine expects a standardized `TransactionSchema`:
* `user_id`: Unique identifier (UUID)
* `amount`: Transaction value in GBP
* `mcc`: ISO-18245 Merchant Category Code
* `timestamp`: ISO-8601 format

---

## 4. Scalability & Future Roadmap
While the MVP runs as a Python-based engine, the architecture is designed for horizontal scaling:
* **Ingestion:** Transitioning to **Apache Kafka** for real-time stream processing.
* **Storage:** Implementing **HDFS** or **AWS S3** for long-term audit logs.
* **Backend:** Porting core detection logic to **Go (Golang)** for low-latency execution (aligned with Monzo's stack).
